Node 基础

Talk is cheap, show me the code

Node

Node优势

Web中场景
  • 静态资源处理
  • 数据库操作
  • 渲染页面 -- CPU快，IO慢。 Web场景是典型IO密集场景
高并发常规应对方法
  • 增加机器数
  • 增加每台机器CPU数 -- 多核
进程

进程： 计算机的程序关于某数据集合上的一次运行活动，是系统进行资源分配和调度的基本单位

多进程： 启动多个进程，快速进行切换

线程

线程： 进程内一个相对独立的、可调度的执行单元，与同属一个进程的线程共享进程资源

多线程：启动一个进程，在一个进程内启动多个线程，多个线程也可以一块执行多个任务

  • Node.js 的单线程

单线程只是针对主进程，IO操作系统底层多线程调度。

单线程并不是单进程

NodeJS性能好，只针对Web这种场景，高并发，密集IO的操作，并不是说所有都比C、Java好

Node环境

环境
  • CommonJS
  • global - 全局
  • process

Node没有BOM DOM 和window

CommonJS
  • 每个文件一个模块，有自己的作用域。
  • 模块内部 module变量代表模块本身
  • module.exports属性代表模块对外接口
Require 规则

  • / 绝对路径 ./ 相对当前文件

  • 支持js , json, node扩展名，不写的话，逐级找

  • 不写路径，则认为是build-in模块，或者各级node_modules第三方模块

Require 特性

  • require的module 被加载的时候执行，加载后缓存 -> 加载一次后，就执行内存中的结果

  • 一旦一个模块被循环依赖，就只输出已经执行的部分，还未执行的不会输出。

Path
const { join } = require('path');
const { normalize } = require('path');
const { resolve } = require('path'); // 相对路径解析成绝对
const { basename, dirname, extname } = require('path'); // 文件名 目录 扩展名
const { parse, format } = require('path'); // 解析 
const { sep, delimiter, win32, posix} = require('path');
  • dirname、filename 总是返回文件的绝对路径
  • process.cwd总是返回执行node命令所在的文件夹

对于./ 或../

  • require方法中总是相对于文件所在的文件夹
  • 在其他地方和process.cwd一样，相对node启动文件夹
Buffer
  • 直接在global上，使用频率很高
  • Buffer 用于处理二进制数据流
  • 实例类似整数数组，大小固定
  • C++ 代码在V8 堆外分配的物理内存
