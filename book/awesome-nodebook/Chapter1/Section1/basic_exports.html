Exports

export default, export/import, module.exports, exports的关系

Conclusion

先来个归类总结，下面再看其他。

  • require => node 和 es6 都支持的引入
  • export / import => 只有es6支持的导入导出
  • module.exports / exports => 只有node支持的导出

Node

Node模块系统遵从CommonJS规范

CommonJS定义的模块分为: 模块标识(module)、模块定义(exports) 、模块引用(require)

exports和module.exports

在一个node执行一个文件时，会给这个文件内生成一个 exports和module对象， 而module又有一个exports属性。他们之间的关系如下图，都指向一块{}内存区域。

exports = module.exports = {}

举个例子：

// common.js

let a = 100;

console.log(module.exports);
console.log(exports);

exports.a = 200;
exports = 'other';

// test.js

const a = require('./common.js');

console.log(a);

▶ node test.js
{}
{}
{ a: 200 }

从上面可以看出，其实require导出的内容是module.exports的指向的内存块内容，并不是exports的。 简而言之，区分他们之间的区别就是 exports 只是 module.exports的引用，辅助后者添加内容用的

所以exports只是辅助module.exports的，真正到处去的还是后者。

使用的时候尽量使用module.exports 即可规避这个问题。

ES6模块

相对清晰一些。

export 和 export default
  • export与export default均可用于导出常量、函数、文件、模块等
  • 在一个文件或模块中，export、import可以有多个，export default仅有一个
  • 通过export方式导出，在导入时要加{ }，export default则不需要
  • export能直接导出变量表达式，export default不行。
// es6_module.js

'use strict'

export const a = 100;

export const aplus = () => {
    console.log('this is a');
}

function b() {
    console.log('this is b');
}

export { b };

const c = 200;

export default c;

//export defult const c = 200;// 这里不能写这种格式。

// test_es6.js
import { aplus, b } from "./es6_module"; //导出了 export 方法 
import c from "./es6_module";  //导出了 export default 


import * as esModule from './es6_module' //as 集合成对象导出

aplus();

b();

console.log(c);

esModule.aplus();

console.log(esModule.c); // undefined , 因为  as 导出是 把 零散的 export 聚集在一起作为一个对象，而export default 是导出为 default属性。

console.log(esModule.default); // 100
