正则表达式

Posted on 2018-01-22   |   In FrontEnd

Overview

进行移动端开发的时候，需要各种对于UA的判断，从而进行机型或系统的匹配进行特殊的页面处理。以下面为例：

// 判断是微信、QQ或微博中打开
var isQQorWeChatorWeibo = /micromessenger/.test(ua) || /qq\//.test(ua) || /weibo/.test(ua);
// 判断是否为ios设备
var isIOS = /(iPhone|iPad|iPod|iOS)/i.test(ua);

// 判断是否为android设备
var isAndroid = /(Android)/i.test(ua);

什么是正则表达式

所以这里就需要一些对正则表达式的知识，这里自己做一下简单的总结。

首先什么是正则表达式，很多人大学的时候应该都学过。

正则表达式，又称规则表达式。（英语：Regular Expression，在代码中常简写为regex、regexp或RE），计算机科学的一个概念。正则表达式通常被用来检索、替换那些符合某个模式(规则)的文本。 — 百度百科

英文释义大概是这样：

A regular expression (regex or regexp for short) is a special text string for describing a search pattern.
You are probably familiar with wildcard notations such as *.txt to find all text files in a file manager. The regex equivalent is ^.*.txt$.

工具

给大家介绍一个常用的reg测试网站，https://regexr.com/

常用

/g, /i, /m 说明

  • /g 表示该表达式将用来在输入字符串中查找所有可能的匹配，返回的结果可以是多个。如果不加/g最多只会匹配一个

  • /i 表示匹配的时候不区分大小写

  • /m 表示多行匹配，什么是多行匹配呢？就是匹配换行符两端的潜在匹配。影响正则中的^$符号

例子/g：

<script type="text/javascript">
    str = "jason li (111)"+ "jason li (222)";  
    ret = str.match(/jason/);    //没有加/g  
    console.log(ret);            //显示一个jason  

    ret = str.match(/jason/g);   //加了/g  
    console.log(ret);            //显示为jason, jason 
</script>

例子/i：

<script type="text/javascript">
    str = "jason Li (111)"+ "jason li (222)";  
    ret = str.match(/li/);    //没有加/i  
    console.log(ret);            //li  

    ret = str.match(/li/ig);   //加了/gi  
    console.log(ret);            //显示为Li, li
</script>

/m用的比较少，用的时候再查吧。

常见匹配规则

\ 做为转意，即通常在"\"后面的字符不按原来意义解释，如/b/匹配字符"b"，当b前面加了反斜杆后/\b/，转意为匹配一个单词的边界。 

-或- 
对正则表达式功能字符的还原，如"*"匹配它前面元字符0次或多次，/a*/将匹配a,aa,aaa，加了"\"后，/a\*/将只匹配"a*"。 

^ 匹配一个输入或一行的开头，/^a/匹配"an A"，而不匹配"An a"

$ 匹配一个输入或一行的结尾，/a$/匹配"An a"，而不匹配"an A"

* 匹配前面元字符0次或多次，/ba*/将匹配b,ba,baa,baaa 

+ 匹配前面元字符1次或多次，/ba*/将匹配ba,baa,baaa 

? 匹配前面元字符0次或1次，/ba*/将匹配b,ba 

(x) 匹配x保存x在名为$1...$9的变量中 

x|y 匹配x或y 

{n} 精确匹配n次 

{n,} 匹配n次以上 

{n,m} 匹配n-m次 

[xyz] 字符集(character set)，匹配这个集合中的任一一个字符(或元字符) 

[^xyz] 不匹配这个集合中的任何一个字符 

[\b] 匹配一个退格符 

\b 匹配一个单词的边界 

\B 匹配一个单词的非边界 

\cX 这儿，X是一个控制符，/\cM/匹配Ctrl-M 

\d 匹配一个字数字符，/\d/ = /[0-9]/ 

\D 匹配一个非字数字符，/\D/ = /[^0-9]/ 

\n 匹配一个换行符 

\r 匹配一个回车符 

\s 匹配一个空白字符，包括\n,\r,\f,\t,\v等 

\S 匹配一个非空白字符，等于/[^\n\f\r\t\v]/ 

\t 匹配一个制表符 

\v 匹配一个重直制表符 

\w 匹配一个可以组成单词的字符(alphanumeric，这是我的意译，含数字)，包括下划线，如[\w]匹配"$5.98"中的5，等于[a-zA-Z0-9]

\W 匹配一个不可以组成单词的字符，如[\W]匹配"$5.98"中的$，等于[^a-zA-Z0-9]

上面这个从一个网站趴下来的，当做手册了。

常用实例

"^\\d+$"　　//非负整数（正整数 + 0） 

"^[0-9]*[1-9][0-9]*$"　　//正整数 

"^((-\\d+)|(0+))$"　　//非正整数（负整数 + 0） 

"^-[0-9]*[1-9][0-9]*$"　　//负整数 

"^-?\\d+$"　　　　//整数 

"^\\d+(\\.\\d+)?$"　　//非负浮点数（正浮点数 + 0） 

"^(([0-9]+\\.[0-9]*[1-9][0-9]*)|([0-9]*[1-9][0-9]*\\.[0-9]+)|([0-9]*[1-9][0-9]*))$"　　//正浮点数

"^((-\\d+(\\.\\d+)?)|(0+(\\.0+)?))$"　　//非正浮点数（负浮点数 + 0） 

"^(-(([0-9]+\\.[0-9]*[1-9][0-9]*)|([0-9]*[1-9][0-9]*\\.[0-9]+)|([0-9]*[1-9][0-9]*)))$"　　//负浮点数

"^(-?\\d+)(\\.\\d+)?$"　　//浮点数 

"^[A-Za-z]+$"　　//由26个英文字母组成的字符串 

"^[A-Z]+$"　　//由26个英文字母的大写组成的字符串 

"^[a-z]+$"　　//由26个英文字母的小写组成的字符串 

"^[A-Za-z0-9]+$"　　//由数字和26个英文字母组成的字符串 

"^\\w+$"　　//由数字、26个英文字母或者下划线组成的字符串 

"^[\\w-]+(\\.[\\w-]+)*@[\\w-]+(\\.[\\w-]+)+$"　　　　//email地址 

"^[a-zA-z]+://(\\w+(-\\w+)*)(\\.(\\w+(-\\w+)*))*(\\?\\S*)?$"　　//url
