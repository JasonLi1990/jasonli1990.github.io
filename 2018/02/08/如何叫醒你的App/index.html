如何叫醒你的App

Posted on 2018-02-08   |   In FrontEnd

概况

在做H5活动的时候，经常遇到这样的需求，就是要把页面分享到微信、QQ、微博等第三方社交应用中，然后再跳回来原来的app中，作为一种拉新或者导流的策略，然而这个时候，问题来了，从这个圈子跳出去了，还跳的回来么，这就好比生活，在一个圈子里转来转去，跃跃欲试，岂不知跳出去容易，跳回来难啊。

目标

废话不多说，先上一张我们分享页的截图。我们要做到的事情就是，点击“火星下载”或者“打开App”按钮，如果用户已经安装了App，则根据业务跳转到相应的Native页面；如果用户没有安装该应用，则跳到AppStore或者应用市场去下载我们的App。

设计

首先说下苹果系统，在ios9以前，唤起app的方式和安卓一样，都是通过scheme进行唤起。这种方式虽然可自定程度很高，能够巧妙地实现很多跳转，但弊端也很明显：我们只能通过scheme://example这种格式的链接来实现跳转，而且现在苹果还对这种方式的跳转加了一个提示框：“是否打开XXX”。对于对Web和原生App交互的场景需求量很大的产品来说，这样的跳转方式显然是步骤冗杂的，用户体验并不好。

而在ios9以后，苹果官方提供了一种唤起app的方式，也就是传说中的universal link，直接跳转指定app不在页面做任何停留。

Universal Link

通过与客户端配合，

  • 在苹果开发者网站中打开需要使用Universal Link功能的App中的Associated Domains

  • 上传apple-app-site-association到服务器根目录下

  • 在AppDelegate中实现相应的方法

apple-app-site-association文件格式如下：

{
    "applinks": {
        "apps": [],
        "details": [
            {
                "appID": "XXX",
                "paths": [ "**/**/*" ] 
            }
        ]
    }
}

几种方式

微信

微信是大多数app做分享的一种方式，但是微信的各种屏蔽策略确为app的分享跳转产生了诸多阻碍，在微信6.6版本之前，ios9提供universal link而微信还没有屏蔽universal link这段时间，是最幸福的，分享页面可以通过点击按钮第一时间跳转到app进行倒流，没有安装app的也会自然的去下载页进行下载，但是新版本微信更新后就不行了，so，不管是ios还是android，目前的跳转策略都是跳到应用宝进行跳转。

哦，微信还屏蔽了itunes的链接（微笑）

QQ

QQ目前还是支持universal link的，android平台也支持scheme唤起方式，

微博

微博目前还支持 universal link唤起，我们只需要考虑未下载的情况。

在IOS下，微博不支持打开应用宝链接，所以需要引导用户通过safari打开链接，然后从safari进行跳转操作。

safari

Apple自家浏览器，肯定是完全支持的，universal link和itunes链接，都支持。按照正常逻辑进行跳转即可。

唤起App

 var ifr = document.createElement('iframe');
if (targetUrl) {
    ifr.src = targetUrl;
}
else {
    ifr.src = 'scheme://xxx';
}

ifr.style.display = 'none';
document.body.appendChild(ifr);
if (stayCurrentPage !== true) {
    window.setTimeout(function () {
        if (commonObj.browser.isAndroid || (!commonObj.browser.isAndroid && (!document.webkitHidden || !document.hidden))) {
            var a = +new Date;
            if (a - g < 1550) {
                document.body.removeChild(ifr);
                if (downloadUrl && typeof downloadUrl === 'string' && downloadUrl.length > 0) {
                    window.location.href = downloadUrl;
                }
            }
        }
    }, 1500);
}
setTimeout(function () {
    document.body.removeChild(ifr);
}, 3000);

看唤起是不是成功了，需要自己来计算时间，因为要是唤起成功了， setInterval的时间就会变慢，经我测试，已经用不到这种方法了，只需要使用 document.hidden||document.webkitHidden就可以，兼容性还不错
