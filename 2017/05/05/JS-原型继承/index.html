JS-原型继承

Posted on 2017-05-05   |   In FrontEnd

概述

原型和闭包是JS的两个难点，最近碰到了原型继承的概念，正好在这里总结一下。

既然要实现继承，就一定要有一个父类。

// 定义一个父类 Father
function Father(name) {
    // 属性
    this.name = name || 'father';
    // 实例方法
    this.say = function() {
        console.log('My name is ' + this.name);
    }
}

// 原型方法
Father.prototype.work = function(job) {
    console.log(this.name + " 's job " + job);
}

原型链继承

function Son1() {}

Son1.prototype = new Father();
Son1.prototype.name = 'son1';

var son1 = new Son1();
console.log('原型链继承');
console.log(son1.name);
console.log(son1.work('student'));
console.log(son1.say());
console.log(son1 instanceof Father);
console.log(son1 instanceof Son1);

特点：

  • 非常纯粹的继承关系，实例是子类的实例，也是父类的实例
  • 父类新增原型方法/原型属性，子类都能访问到
  • 简单，易于实现
    缺点：
  • 要想为子类新增属性和方法，必须要在new Animal()这样的语句之后执行，不能放到构造器中
  • 无法实现多继承
  • 来自原型对象的引用属性是所有实例共享的（详细请看附录代码： 示例1）
  • 创建子类实例时，无法向父类构造函数传参

构造继承

使用父类的构造函数来增强子类实例，等于是复制父类的实例属性给子类（不用原型）

function Son1(name) {
    Father.call(this);
    this.name = name || 'Son1'
}

var son1 = new Son1();
console.log('构造继承');
console.log(son1.name);
// console.log(son1.work('student')); // 不行，不继承原型属性方法
console.log(son1.say());
console.log(son1 instanceof Father); // false 
console.log(son1 instanceof Son1); // true

特点：

  • 解决了1中，子类实例共享父类引用属性的问题
  • 创建子类实例时，可以向父类传递参数
  • 可以实现多继承（call多个父类对象）

缺点：

  • 实例并不是父类的实例，只是子类的实例
  • 只能继承父类的实例属性和方法，不能继承原型属性/方法
  • 无法实现函数复用，每个子类都有父类实例函数的副本，影响性能

实例继承

为父类实例添加新特性，作为子类实例返回

function Son1(name){
  var instance = new Father();
  instance.name = name || 'Son1';
  return instance;
}

// Test Code
var son1 = new Son1();
console.log(son1.name);
// console.log(son1.sleep());
console.log(son1 instanceof Father); // true
console.log(son1 instanceof Son1); // false

特点：

  • 不限制调用方式，不管是new 子类()还是子类(),返回的对象具有相同的效果

缺点：

  • 实例是父类的实例，不是子类的实例
  • 不支持多继承

拷贝继承

function Son1(name) {
    var father = new Father();
    for (var p in father) {
        Son1.prototype[p] = father[p];
    }

    Son1.prototype.name = name || 'Son1';
}

var son1 = new Son1();
console.log(son1.name);
// console.log(son1.sleep());
console.log(son1 instanceof Father); // false
console.log(son1 instanceof Son1); // true

特点：

  • 支持多继承

缺点：

  • 效率较低，内存占用高（因为要拷贝父类的属性）
  • 无法获取父类不可枚举的方法（不可枚举方法，不能使用for in 访问到）

组合继承

通过调用父类构造，继承父类的属性并保留传参的优点，然后通过将父类实例作为子类原型，实现函数复用

function Son1(name) {
    Father.call(this);
    this.name = name || 'Son1';
}

Son1.prototype = new Father();

var son1 = new Son1();
console.log('组合继承');
console.log(son1.name);
console.log(son1.work('student'));
console.log(son1.say());
console.log(son1 instanceof Father); // true
console.log(son1 instanceof Son1); // true

特点：

  • 弥补了方式2的缺陷，可以继承实例属性/方法，也可以继承原型属性/方法
  • 既是子类的实例，也是父类的实例
  • 不存在引用属性共享问题
  • 可传参
  • 函数可复用

缺点：

  • 调用了两次父类构造函数，生成了两份实例（子类实例将子类原型上的那份屏蔽了）

寄生组合继承

通过寄生方式，砍掉父类的实例属性，这样，在调用两次父类的构造的时候，就不会初始化两次实例方法/属性，避免的组合继承的缺点

function Son1(name) {
    Father.call(this);
    this.name = name || 'Son1';
}

(function(){
    var Super = function(){};
    Super.prototype = Father.prototype;
    Son1.prototype = new Super();
})();

// Son1.prototype = new Father();

var son1 = new Son1();
console.log('组合继承');
console.log(son1.name);
console.log(son1.work('student'));
console.log(son1.say());
console.log(son1 instanceof Father); // true
console.log(son1 instanceof Son1); // true

特点：

  • 堪称完美

缺点：

  • 实现较为复杂
