CSS-多行省略问题

Posted on 2017-12-16   |   In FrontEnd

Overview

平时写H5项目或者活动的时候一定会遇到这样一个小问题，也就是文案过长需要截断的情况，以我们平台的一个例子为例，我们常做的榜单活动里，会涉及主播名称过长的时候，需要一行显示，则需要做相应的截断，平时的常见页面，用简单的css属性就可以搞的定。如下：

通用方案

html:

<article>
    <section class="content">
        <section class="wrapper">
            <p class="text-nowrap">你好这里好长好长好长好长好长好长好长好长好长好长好长</p>
        </section>
    </section>
</article>

css：

h2 {
    display: flex;
    align-items: center;
    justify-content: center;
}
.content {
    position: relative;
    display: block;
    width: 100px;
    border: 2px solid #eee;
    margin: 0 auto;
}

效果如下：

Flex布局

css：

<style type="text/css">
    h2 {
        display: flex;
        align-items: center;
        justify-content: center;
    }
    .content {
        position: relative;
        display: flex;
        width: 100px;
        border: 2px solid #eee;
        margin: 0 auto;
    }

    .content .wrapper {
        flex: 1;
    }

    .text-nowrap {
        overflow: hidden;
        white-space: nowrap;
        text-overflow: ellipsis;
    }
</style>

这个时候会发现变成了下面这个样子：

这是因为wrapper没有限定宽度造成text-overflow: ellipsis失效，给wrapper加上宽度即可。

多行省略

下面问题又来了，如果想显示成多行文案，最后一行再省略怎么办，这也是在css文本省略问题中一个经常遇到的问题。

其实解决方案也很简单，在text-nowrap中加入以下属性，去掉white-space: nowrap属性即可。

display: -webkit-box;
-webkit-line-clamp: 1;
-webkit-box-orient: vertical;

-webkit-line-clamp: 1; 表示想显示几行，比如这我设置成2；

但有的时候也会失效，需要添加word-break: break-all;属性。

综上

综上，统一解决方案可以这样：

.text-nowrap {
    overflow: hidden;
    text-overflow: ellipsis;
    display: -webkit-box;
    -webkit-line-clamp: 1;
    -webkit-box-orient: vertical;
    word-break: break-all;
}
