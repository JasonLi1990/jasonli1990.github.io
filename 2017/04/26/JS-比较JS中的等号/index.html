JS-比较JS中的==与===

Posted on 2017-04-26   |   In FrontEnd

概述

JS中一般判断相等时有两种运算符，等于运算符==和恒等运算符===。

和php等语言中一样，一般给人的第一印象都是，恒等运算符===给人一种严格等于的感觉，==属于不严格的比较，但是JS中这两个符合具体有什么区别呢，今天突然想到这个问题，之前看了一些资料，在这里总结一下。

是否类型转换

简单来说，

“==” 运算符是先做完需要的类型转换后再比较相等

“===” 恒等运算符不会做类型转换，所以如果不是相同类型做比较的话， === 会直接return false。

在这种情况下， === 比 == 更快，并且可能返回不同的结果。在其他情况下两个运算符表现一样。

详述

详细一点来说，根据操作对象的类型不同可以分为一下三种情况：

  • String，Number等基础类型间的比较， == 和 === 是有区别的。

不同类型间的比较： == 会尝试将两者转换为同一类型后比较值是否相等， === 则是类型不同则返回false。

相同类型间的比较是直接的值比较，两者结果一样

String("1") == Number(1) //true
String("1") === Number(1) //false

  • 对于Array,Object等高级类型，”==” 和 “===” 进行的是’地址’比较，两者是没有区别的。

  • 基础类型与高级类型相比较：”==” 会尝试将高级类型转换为基础类型后进行值比较。 “===” 会直接返回false。

== 对操作值进行隐式类型转的规则

  • 如果一个操作值为布尔值，则在比较之前先将其转换为数值
  • 如果一个操作值为字符串，另一个操作值为数值，则通过Number()函数将字符串转换为数值
  • 如果一个操作值是对象，另一个不是，则调用对象的valueOf()或* toString()方法，得到的结果按照前面的规则进行比较
  • null与undefined是相等的
  • 如果一个操作值为NaN，则相等比较返回false
  • 如果两个操作值都是对象，则比较它们是不是指向同一个对象

注意

NaN不与任何值相等。

如果一个操作值是对象，另一个是基础类型，那么会先尝试对对象调用valueOf()，如果不能返回基础类型，则调用toString。

[1,2,3] == 1 //false
[1] == 1 //true
[1] === 1 //false


true == 1  // true, 1==1
true == 2  // false , true转换为1， 1==2
true == !!2// true, !2 会将数字2转换为boolean值， !2 = false, !!2 = true

=== 对操作值进行隐式类型转的规则

  • 如果类型不同，就[不相等]
  • 如果两个都是数值，并且是同一个值，那么[相等]；(！例外)的是，如果其中至少一个是NaN，那么[不相等]。（判断一个值是否是NaN，只能用isNaN()来判断）
  • 如果两个都是字符串，每个位置的字符都一样，那么[相等]；否则[不相等]。
  • 如果两个值都是true，或者都是false，那么[相等]。
  • 如果两个值都引用同一个对象或函数，那么[相等]；否则[不相等]。
  • 如果两个值都是null，或者都是undefined，那么[相等]。
