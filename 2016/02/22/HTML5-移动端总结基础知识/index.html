CSS pixels and Device pixels

  • CSS pixels: 浏览器使用的抽象单位
  • device pixels： 显示屏幕最小物理单位，每个dp包含自己的颜色和亮度
  • CSS像素为设备无关像素（device-independent像素），DIPs。
  • 标准显示度下，一个css像素对应一个设备像素。
  • Retina屏幕下会有成倍显示

Web中用device-pixel-ratio解决，js中window.devicePixelRatio可以获得相同比率，但浏览器支持有限。

公式：dpr(window.devicePixelRatio) = 物理像素/dips
其中dip为设备独立像素，dip可以用来辅助区分视网膜设备还是非视网膜设备

视网膜屏上，屏幕物理像素640px，独立像素还是320px，所以dpr为2

经总结 —> 1 CSS pixels = （devicePixelRatio）^2 device pixels （^2是平方的意思，devicePixelRatio即dpr）

当通过met改变viewport时候，

设置了视区宽度为设备宽度，所以对于iphone4或4s，即为320px。

PPI/DPI

PPI，有时也叫DPI，表示的是每英寸所拥有的像素（pixel）数目，数值越高，即代表显示屏能够以越高的密度显示图像。（注：这里的像素，指的是device pixels）

手机屏幕的对角线等效像素，然后处以对角线（我们平常所说的手机屏幕尺寸就是说的手机屏幕对角线的长度），就可以得到PPI了

密度决定比例

  • 一个设备表面上存在的像素数量，它通常以每英寸有多少像素来计算（PPI）
  • Retina 概念，由Apple提出，当一个显示屏像素密度超过300ppi时，人眼就无法区分

PPI在120-160之间的手机被归为低密度手机，160-240被归为中密度，240-320被归为高密度，320以上被归为超高密度（Apple给了它一个上流的名字——retina）
